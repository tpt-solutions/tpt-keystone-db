TPT Meridian (Next-Gen Geospatial Engine)
The "PostGIS Reimagined"
PostGIS is a triumph of open-source engineering, but it carries 25 years of architectural debt. It was built on top of the GEOS C++ library and Postgres's row-oriented storage, forcing complex geometries through an 8KB page system designed for simple text fields. Spatial joins at scale require expensive R-Tree traversals through a buffer pool that doesn't understand geometry. Real-time tracking of millions of moving objects (autonomous vehicles, drones, IoT sensors) is essentially impossible because every position update triggers a full row rewrite and WAL entry. The result is a system that works beautifully for static maps but breaks down under the demands of modern, dynamic, planetary-scale geospatial applications.
TPT Meridian is a ground-up reimagining of geospatial data management, built as a native module within the TPT Keystone storage engine. Instead of treating geometry as an opaque blob, Meridian understands spatial data at the byte level. It abandons the legacy R-Tree in favor of hierarchical discrete global grid systems (S2 and H3), which partition the planet into a mathematically perfect hierarchy of cells—enabling distributed spatial queries that scale horizontally across thousands of nodes. It natively supports 4D spatiotemporal data, treating time as a first-class dimension alongside X, Y, and Z, so a query like "find all drones that were within 500 meters of this coordinate between 2:00 and 2:05 PM" executes as a single, optimized index scan. Furthermore, Meridian includes GPU-accelerated spatial joins via compute shaders, turning operations that take minutes in PostGIS into sub-second operations.
By releasing TPT Meridian as an Apache 2.0 public good, we unlock geospatial innovation for everyone. Urban planners in developing nations, open-source robotics projects, climate researchers, and indie game developers all gain access to planetary-scale spatial analytics without licensing fees. Meridian also unifies raster and vector data in a single storage model, eliminating the painful dichotomy that forces teams to maintain separate systems for satellite imagery and feature data. It becomes the foundation for a new generation of open-source location intelligence, where the geography of our world is treated as a shared, programmable commons.
Tech Stack:
Core Geometry Engine: Rust with a custom, memory-safe computational geometry library (replacing GEOS).
Spatial Indexing: Native S2 Geometry and Uber H3 integration for hierarchical, distributed spatial partitioning.
GPU Acceleration: wgpu / CUDA compute shaders for massively parallel spatial joins and raster operations.
Spatiotemporal Storage: Custom 4D space-time partitioning built into the Strata storage engine.
Raster/Vector Fusion: Unified storage model supporting both vector geometries and multi-spectral raster tiles in a single query path.
Standards Compliance: Full OGC Simple Features and SQL/MM Spatial compatibility for drop-in migration from PostGIS.
