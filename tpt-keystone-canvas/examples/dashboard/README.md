# TPT Canvas Dashboard (demo)

First-ever browser verification of the `tpt-keystone-canvas` crate. Mounts all six
`Canvas.*` components against a live `tpt-keystone` node.

## Quick start (no database needed)

A zero-dependency mock of the Canvas HTTP bridge ships with the demo, so
you can see the dashboard render with seeded data immediately:

```bash
# Terminal 1 — seeded mock of POST /query + GET /schema on :5435
npm run mock

# Terminal 2 — Vite dev server in mock mode (points at the mock)
npm run dev:mock
```

Open the printed `http://localhost:5173` URL. Every card runs real SQL
through `use_keystone_query` against the mock (which pattern-matches the
demo's SQL and returns seeded rows), so the wiring is verified end-to-end
without a live `tpt-keystone` node.

## Against a real node

```bash
# 1. Start a node: `cargo run` in tpt-keystone (Canvas HTTP on 5435).
# 2. Build the WASM package (emits ./pkg, consumed by Vite as an ES module).
npm run build:wasm

# 3. Start the dev server (defaults to http://localhost:5435).
npm run dev
```

Open the printed URL. Each card runs real SQL via `use_keystone_query`
against the node; the Map card renders a kernel-density heatmap, the Graph
card runs a force-directed layout, the Document card edits JSON in place via
`jsonb_set`, and the Agent Monitor combines a DOM event timeline with a
Canvas2D latency bar chart.

## Point at a different node

```bash
TPT_HTTP_ADDR_BASE=http://10.0.0.5:5435 TPT_FLUX_WS_ADDR=ws://10.0.0.5:5434 npm run dev
```

## Files

- `index.html`, `src/main.ts`, `src/tpt-canvas.d.ts` — the demo app
- `vite.config.ts` — dev server + `mock` mode (defaults base URLs to the mock)
- `mock-server.mjs` — zero-dep seeded Canvas HTTP bridge for offline runs
- `pkg/` — generated by `npm run build:wasm` (wasm-bindgen `--target web`)

## Expected schema

The demo assumes tables/functions the node already exposes (Chronos rollups,
Plexus `nodes`/`edges`, Prism `vector_search`, Canopy `documents`, Mirror
`mirror_session_events`/`mirror_agent_metrics`). The mock server covers the
same names with sample data. If a component's SQL finds no rows it simply
renders empty — no crash.
