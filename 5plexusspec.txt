TPT Plexus (Native Graph Engine)
The "Neo4j Reimagined"
Graph data is the hidden architecture of the modern world: social networks, fraud detection rings, supply chain dependencies, and AI knowledge graphs. But current graph databases are either locked behind exorbitant proprietary licenses or bolted onto relational databases as an afterthought. The fundamental problem is that relational databases store data in rigid rows, but graph algorithms need to traverse edges. When you try to do a 5-hop "friend-of-a-friend" traversal in a traditional SQL database, it results in a nightmare of expensive self-joins that brings the system to its knees.
TPT Plexus makes graph traversal a native, first-class operation of the storage engine. Instead of treating edges as rows in a table, Plexus implements a native adjacency list storage format optimized specifically for graph algorithms. Each vertex stores direct memory pointers to its neighbors, allowing traversals to execute in microseconds without expensive join operations. Plexus natively supports property graphs (where both vertices and edges can have properties), multi-relational edges, and bidirectional traversal. Furthermore, it includes native, highly optimized implementations of common graph algorithms—such as shortest path, PageRank, community detection, and connected components—executed directly within the storage engine.
By releasing TPT Plexus under the Apache 2.0 license, we democratize graph analytics for the entire open-source ecosystem. Social networks, fraud detection systems, and academic research projects can leverage native graph traversal without paying licensing fees or maintaining a separate, siloed database. Crucially, Plexus supports hybrid queries, allowing you to filter vertices by structured relational properties, traverse deep relationship paths, and return the results alongside standard SQL data—all in a single, unified query.
Tech Stack:
Core Engine: Rust with native adjacency list storage optimized for zero-copy graph traversals.
Graph Model: Property graph model supporting vertices, edges, and arbitrary properties on both.
Query Language: GQL (Graph Query Language) compatibility, alongside SQL extensions for intuitive graph traversal syntax.
Graph Algorithms: Native, parallelized implementations of shortest path, PageRank, community detection, and connected components.
Indexing: Native edge indexing and triangle indexing for lightning-fast neighbor lookups and relationship filtering.
