TPT Prism (Native Vector & AI Engine)
The "Semantic Spectrum"
The explosion of AI has turned vector search into a critical database workload, but the current solutions are fundamentally compromised. pgvector is a brilliant hack that bolted vector columns onto Postgres, but it forces billion-scale HNSW graphs through a buffer pool designed for relational rows, resulting in catastrophic memory usage. Dedicated vector databases (Pinecone, Weaviate) solve the performance problem but force teams to maintain a completely separate infrastructure stack, duplicating data and creating synchronization nightmares. Vector data is not a niche workload—it is becoming as fundamental as text, and it deserves to be a first-class citizen of the database, not an afterthought.
TPT Prism is a native vector engine built directly into the TPT Keystone storage layer, treating embeddings with the same architectural respect as integers or geometries. At the storage level, Prism implements multiple index types simultaneously—HNSW for high-recall queries, IVF-PQ for memory-constrained billion-scale datasets, and DiskANN for cold-storage vectors—allowing the query planner to automatically select the optimal index based on latency and recall requirements. It leverages hardware-accelerated SIMD instructions (AVX-512, NEON) and optional GPU offloading to compute cosine similarity at wire speed. Crucially, Prism introduces native hybrid search, where a single query can combine vector similarity, BM25 full-text search, structured SQL filters, and Meridian spatial predicates—all resolved in a single pass through the storage engine with zero data movement.
By releasing TPT Prism under the Apache 2.0 license, we eliminate the artificial separation between operational databases and AI infrastructure. Developers no longer need to choose between the rich query capabilities of a relational database and the semantic search power of a vector database—they get both, unified, with ACID transactions and strong consistency. Prism also introduces native quantization at the storage layer, automatically compressing embeddings as they are inserted while preserving recall, dramatically reducing the cost of storing billions of vectors and making semantic search accessible to the entire open-source ecosystem.
Tech Stack:
Core Vector Engine: Rust with SIMD-first design, utilizing std::arch for AVX-512/NEON acceleration.
Index Types: Native implementations of HNSW, IVF-PQ, IVF-SQ, and DiskANN, with automatic index selection.
Hardware Acceleration: Optional CUDA / ROCm offloading for massive batch similarity computations.
Hybrid Query Planner: Custom cost-based optimizer that fuses vector, full-text (BM25), structured, and spatial predicates.
Quantization: Native Product Quantization (PQ), Scalar Quantization (SQ), and Binary Quantization at the storage layer.
Distributed Sharding: Consistent hashing of vector spaces for horizontal scaling across Keystone nodes.
