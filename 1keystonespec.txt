TPT Keystone (The Next-Gen Relational Database)
The "Post-Modern Postgres"
The open-source world relies on PostgreSQL as the foundational bedrock for almost every modern application, but its 1996-era architecture is fundamentally hostile to modern infrastructure. Postgres was designed for spinning hard drives, single-core CPUs, and monolithic servers. Its process-per-connection model causes catastrophic memory exhaustion in serverless environments, its 8KB page size causes massive write amplification on modern NVMe drives, and its monolithic compute-storage coupling makes horizontal scaling in Kubernetes a nightmare. Developers are forced to bolt on fragile connection poolers, battle autovacuum bloat, and rely on proprietary cloud forks just to make it scale.
TPT Keystone is a ground-up, cloud-native relational database written in Rust, designed specifically for the realities of 2026 hardware. It features a stateless, thread-per-core compute layer that multiplexes connections asynchronously, allowing it to handle millions of concurrent sessions with a fraction of the memory footprint of Postgres. Its disaggregated architecture separates compute from data, storing the source of truth in S3-compatible object storage while utilizing a local, NVMe-optimized Log-Structured Merge (LSM) tree for blazing-fast, zero-copy asynchronous I/O via io_uring. Crucially, TPT Keystone speaks the exact PostgreSQL wire protocol, meaning every existing ORM, driver, and migration tool works out of the box, while its WebAssembly-based extension system ensures that custom plugins are memory-safe and completely crash-proof.
By releasing TPT Keystone under the Apache 2.0 license, we give the community a true, next-generation relational database that they fully own. It eliminates the need to rely on heavily managed, proprietary cloud databases just to get horizontal scaling and serverless compatibility. It provides a mathematically modern MVCC garbage collection system that completely eliminates the dreaded "vacuum" maintenance windows, securing the foundation of the open-source web for the next thirty years without the architectural debt of the past.
Tech Stack:
Core Engine: Rust (for memory safety, zero-cost abstractions, and modern async concurrency).
Networking/Compute: Tokio combined with a Thread-Per-Core architecture to eliminate mutex contention and process forking.
Storage Engine: Custom NVMe-optimized LSM-Tree utilizing io_uring for zero-copy asynchronous disk I/O.
Disaggregated Storage: Integration with S3-compatible object storage via the AWS SDK for Rust, with a local NVMe/SPDK caching layer.
Extensions: Wasmtime embedded to safely execute user-defined database functions in a sandboxed environment.
Compatibility: 100% implementation of the PostgreSQL Wire Protocol (pgwire) for drop-in compatibility.
